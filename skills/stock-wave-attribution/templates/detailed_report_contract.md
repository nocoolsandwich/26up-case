# 股票波段归因详细表格合同

本模板用于项目内 `stock-wave-attribution` orchestrator 的正式输出契约，默认对齐“五洲新春”案例颗粒度。

固定结构：

1. `基础信息`
   - 必须包含 `报告时间`
   - 必须包含 `一句话逻辑`
2. 仅分析 `涨幅 Top2` 波段
3. 每个波段单独成段，且一级标题固定为 `波段 Wn`
4. 每个波段下二级标题固定为：
   - `证据原文`
   - `量价验证表`
   - `概念联动验证表`
   - `结论与置信度表`
   - `综合裁决`

补充规则：

- 正式服务链默认不依赖 `ChatGPT 联网归因`
- 服务模式固定按两阶段落盘：
  1. 先写 `data/service_tasks/<task-id>/agent_rerank/summary.json`
  2. 再按各波段 `rough_chunks/chunk_*.md` 做 `100选3-5` 粗排
  3. 最后把最终结果写到 `data/service_tasks/<task-id>/agent_rerank/final_selection.json`
  4. 再执行正式报告收口
- 报告中的 `粗排新闻来源分布` 必须来自波段粗排候选池，而不是最终入选证据

表头固定如下：

每个波段标题下必须先给出波段概览：

- `区间`
- `涨幅`
- `波段审查`
- `粗排新闻来源分布`
- `一句话逻辑`

## 证据原文

每条证据固定为：

- `#### 证据 N`
- `时间 / 来源 / 标题 / 链接`
- 按 `发布时间升序` 排列，越早的强信号越靠前
- fenced code block 包裹的整段数据库原文
